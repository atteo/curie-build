Subsystem · formatter

Formatter

curie fmt rewrites your Java sources to a consistent style — powered by palantir-java-format, which is a fork of google-java-format with reflow improvements.

What it runs

The formatter and its closure (Jackson, Guava, etc.) are resolved from Maven Central the first time you invoke curie fmt; subsequent runs hit the local ~/.m2 cache. There is no separate install step and no plugin to wire in.

Usage

$ curie fmt 
# rewrites every .java file in place to the AOSP style.

Add --check to forbid edits — useful in CI. Curie exits non-zero if any file would change and prints the offending paths.

$ curie fmt --check 
./src/com.example/Greeter.java
error: fmt: one or more files are not correctly formatted.
       Run `curie fmt` (without --check) to fix them.

Source discovery

Every .java file under your project's source roots is included:

  • src/main/java/ — Maven-style production sources
  • src/test/java/ — Maven-style test sources
  • src/<dot-package>/ — flat-package layout
  • tests/<dot-package>/ — flat-package integration tests

Kotlin sources (.kt) are intentionally excluded — they need a separate formatter (ktlint or ktfmt). Adding Kotlin support to curie fmt is on the roadmap.

Style

Curie runs palantir with --aosp, which uses 4-space indentation (Google's default is 2 spaces). The style is otherwise palantir's reflow-friendly variant of google-java-format — line length 120, no manual line-break massaging.

Style is not currently configurable. The argument is that a one-true-style for a given project removes a recurring source of bikeshedding; teams that need something else should keep using their existing formatter and skip curie fmt.

Workspace fan-out

Run curie fmt from a workspace root and Curie fans out across every member in parallel, one java process per member. palantir-java-format is resolved once at the start of the run and the JAR list is shared across the worker threads — concurrent identical resolves would otherwise race on the same ~/.m2 staging files.

$ curie fmt --check # CI in a monorepo 
Workspace . fmt (12 members)
  Formatting      [==============>------] 9/12
    ⠹ jackson-bom-greeter
    ⠹ maven-string-utils
    ⠹ hello-mixed

Ignoring files

Curie passes every discovered .java file to palantir. There is no per-file opt-out. If you need to keep generated or vendored sources untouched, keep them outside the source roots (e.g. under target/ for annotation-processor output, which is automatically excluded).
